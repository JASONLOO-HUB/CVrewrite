# 原始意图提炼：STAR + 产品经理业务周期

## 用途

在正式写简历前，必须先写一版“原始意图提炼”。  
这份提炼既要按 STAR 组织，又要从产品经理的业务周期角度理解项目。

默认使用通用产品经理语义，不要预设项目一定是游戏、内容平台或 AI 工具。

## STAR 的正确用法

这里的 STAR 不是为了写面试故事，而是为了还原项目主线。

### S：Situation

回答：

- 这是一个什么类型的产品？
- 目标用户 / 使用场景是什么？
- 想解决什么问题？
- 为什么会有这个项目？

### T：Task

回答：

- 这个项目最核心的产品任务是什么？
- 第一阶段最想跑通什么？
- 用户最先应该感知到的价值是什么？

### A：Action

回答：

- 产品是如何围绕主线展开的？
- 哪些设计动作真正定义了核心场景 / 业务闭环 / 用户路径？
- 哪些系统是核心，哪些只是辅助？

### R：Result

如果没有真实结果，不要编。

中间分析里最多只能写：

- 预期结果
- 目标效果

最终简历文案里，不要把这些写成真实结果。

## 产品经理业务周期视角

在写 `A` 时，不要只看“有哪些功能”，要按业务周期去理解：

1. 用户是谁，为什么来
2. 第一眼看到什么，如何进入
3. 核心场景中的关键路径是什么
4. 第二层业务闭环是什么
5. 增长、留存或商业化如何嵌进产品
6. 项目最能体现的产品判断是什么

## 推荐输出骨架

原始意图提炼建议至少覆盖下面几项：

- 项目定位
- 核心用户 / 核心场景
- 关键路径 / 关键机制
- 第二层业务闭环（增长 / 留存 / 商业化 / 长期价值）
- 产品主线上的关键设计判断

## 从这份提炼走向简历

只有在这份提炼足够稳定之后，才能继续进入简历改写。

如果这份提炼里仍然答不清：

- 这个项目到底是什么
- 核心场景和关键体验是什么
- 为什么用户会记住它

那就还不能进入最终成稿。
